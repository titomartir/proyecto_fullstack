📌 Información del Proyecto
Campo	Valor

🗄️ Base de Datos PostgreSQL
Instrucción
Genera un script SQL completo que cree la base de datos desde cero con las siguientes tablas.

Esquema requerido
sql
-- Crear base de datos
CREATE DATABASE evaluacion_fullstack;

\c evaluacion_fullstack;

-- Tabla: usuarios (para autenticación)
CREATE TABLE usuarios (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tabla: perfiles (relación 1:1 con usuario)
CREATE TABLE perfiles (
    id SERIAL PRIMARY KEY,
    usuario_id INTEGER UNIQUE NOT NULL,
    nombre VARCHAR(100) NOT NULL,
    apellido VARCHAR(100) NOT NULL,
    edad INTEGER NOT NULL,
    correo VARCHAR(255) NOT NULL,
    telefono VARCHAR(20) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id) ON DELETE CASCADE
);

-- Índices para rendimiento
CREATE INDEX idx_usuarios_email ON usuarios(email);
CREATE INDEX idx_perfiles_usuario_id ON perfiles(usuario_id);

Requerimientos funcionales del backend
1. Configuración (app.js)
Usar express.json() para parsear JSON

Usar cors() para permitir peticiones desde frontend

Registrar rutas: /api/auth y /api/perfil

Middleware para rutas no encontradas (404)

Middleware global de manejo de errores (500)

2. Repositorios (acceso a datos)
usuarioRepository.js

findByEmail(email) → retorna usuario por email

create(email, passwordHash) → crea nuevo usuario

perfilRepository.js

findByUsuarioId(usuarioId) → retorna perfil por usuario_id

create(usuarioId, perfilData) → crea perfil

update(usuarioId, perfilData) → actualiza perfil

Todas las consultas deben usar parámetros preparados ($1, $2)

3. Servicios (lógica de negocio)
authService.js

register(email, password, perfilData)

Verificar si email ya existe

Hashear contraseña con bcrypt (10 rounds)

Crear usuario

Crear perfil automáticamente (relación 1:1)

Generar JWT token

Retornar { token, usuario, perfil }

login(email, password)

Buscar usuario por email

Comparar contraseña con bcrypt

Generar JWT token

Retornar { token, usuario, perfil }

perfilService.js

obtenerPerfil(usuarioId) → retorna perfil o error 404

actualizarPerfil(usuarioId, perfilData)

Validar edad (18-120)

Validar teléfono (8 dígitos)

Actualizar perfil existente

4. Controladores
authController.js

POST /api/auth/register

Validar campos con express-validator

Respuesta exitosa: 201 + { success, token, usuario, perfil }

Error email duplicado: 409

Error validación: 400

POST /api/auth/login

Validar email y password

Respuesta exitosa: 200 + { success, token, usuario, perfil }

Credenciales incorrectas: 401

perfilController.js (todas requieren autenticación)

GET /api/perfil → obtener perfil

PUT /api/perfil → actualizar perfil

5. Middlewares
authMiddleware.js

Verificar header Authorization: Bearer <token>

Validar token con JWT_SECRET

Adjuntar req.usuario con datos decodificados

Retornar 401 si token inválido o no existe

6. Validaciones (express-validator)
Registro (POST /api/auth/register)

email: isEmail()

password: minLength(6)

nombre: notEmpty()

apellido: notEmpty()

edad: isInt({ min: 18, max: 120 })

correo: isEmail()

telefono: matches(/^\d{8}$/)

Login (POST /api/auth/login)

email: isEmail()

password: notEmpty()

Actualizar perfil (PUT /api/perfil)

nombre: notEmpty()

apellido: notEmpty()

edad: isInt({ min: 18, max: 120 })

correo: isEmail()

telefono: matches(/^\d{8}$/)

7. Utilitarios
jwtHelper.js

generateToken(payload) → retorna JWT firmado

verifyToken(token) → retorna payload o lanza error

📝 Notas importantes para la IA generadora
El perfil NO se crea manualmente después del login. Se crea automáticamente durante el registro en una misma transacción (usuario + perfil).

La tabla perfiles tiene dos campos de correo:

correo en perfiles (para mostrar en el perfil)

email en usuarios (para login)

Ambos son independientes y se validan por separado.

El componente Perfil solo debe actualizar, nunca mostrar un formulario vacío para crear.

El registro debe incluir TODOS los campos del perfil en la misma petición.

Usar Angular standalone components (no módulos tradicionales).

Todas las consultas SQL deben ser parametrizadas para evitar SQL injection.

Las contraseñas nunca se almacenan ni se envían en texto plano.

El JWT debe incluir al menos: { id, email }.
